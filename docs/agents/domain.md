# Domain documentation

This is a single-context repository.

Before changing domain behavior, read the root `CONTEXT.md` and relevant decisions in `docs/adr/`. Use the terms defined in `CONTEXT.md` in issues, tests, code, and documentation. If a proposed change conflicts with an ADR, state the conflict explicitly instead of silently replacing the decision.

Create or update domain documentation only when terminology or an architectural decision is concrete enough to preserve.
